# Source Ledger

Every factual statement in the book receives a source label before publication.

## Source labels

| Code | Source type | Use |
|---|---|---|
| T | Author testimony | Steven's memory or direct account |
| R | Repository record | Commits, files, releases, issues, or pull requests |
| P | Platform record | Dated public posts, videos, livestreams, or podcast pages |
| D | Document record | A dated document approved for use |
| X | External source | Independent reporting, review, or public record |

## Confidence labels

- **confirmed** — supported by a dated record or multiple consistent sources
- **author-confirmed** — Steven confirms the event; no independent record is required for first-person memory
- **approximate** — event is accepted but the date or sequence needs refinement
- **disputed** — sources or memories conflict
- **excluded** — not approved for the public book

## Editorial rule

A source supports the event; it does not replace Steven's voice. Editors and models may organize, compare, or flag inconsistencies, but they may not invent facts to fill a gap.
